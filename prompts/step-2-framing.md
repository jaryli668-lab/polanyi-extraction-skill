# 第二步：降维定格引导模板

## 目标
将"附属意识"转化为"集中意识"，把直觉拆解成可操作的步骤。

## 核心原则
> "像解剖青蛙一样，把那个丝滑的直觉，拆解成最笨拙、最机械的步骤。"

## 开场引导
- "我们来把那个瞬间拆解一下。你当时具体做了什么？请用最笨拙的语言描述，比如'我移动了鼠标''我删掉了这个句子'..."
- "像慢动作回放一样，我们一帧一帧地看。第一步、第二步、第三步...分别是什么？"

## 拆解技巧

### 技巧 1：动作动词拆解
**示例**：
- ❌ "我觉得这个标题不好"
- ✅ "我看到这个标题，眉头皱了一下，然后我删掉了前两个字，把后面的结论提到了最前面"

**引导话术**：
- "不要用'我觉得'这种词，告诉我你具体做了什么动作。"
- "你的手移动了哪里？点击了什么？删除了什么？"

### 技巧 2：对比表格
引导用户对比"直觉版本"和"普通版本"的差异：

| 维度 | 普通版本 | 你的直觉版本 |
|------|----------|-------------|
| **具体做了什么** |  |  |
| **身体反应** |  |  |
| **结果** |  |  |

### 技巧 3：三问法
对每个"直觉"，问三个问题：
1. **你看到了什么？**（具体现象）
2. **你想保留什么？**（目标）
3. **你做了什么？**（动作）

**示例**：

**用户**："我就是知道标题要改。"

**你**：
1. "你看到标题时，看到了什么让它'不对'？"
   - "太长？太像说明书？没有情绪？"
2. "你想让读者看完标题后，有什么感觉？"
   - "好奇？兴奋？想点进去？"
3. "所以你做了什么改动？"
   - "加了'我'？改了动词？加了对比？"

## 深层洞察挖掘

从具体动作中，提取规律：

**示例**：

| 具体动作 | 背后的规律 |
|----------|------------|
| 标题加了"我" | 好标题需要有"人" |
| 用"拯救"代替"提高" | 好标题需要有情绪 |
| 用"20 vs 3"的对比 | 好标题需要有冲击力 |

**引导话术**：
- "从这三个动作，你发现什么共同点了吗？"
- "如果要用一句话总结你的直觉规则，会是什么？"

## 判断标准

✅ **成功降维定格的标志**：
- 用户的直觉被拆解成 3-5 个具体步骤
- 每个步骤都是"傻瓜都能照做"的
- 从具体动作中提炼出了规律

❌ **需要继续拆解**：
- 步骤还是太抽象（"要有创意"、"要有感觉"）
- 没有提炼出规律，还是零散的动作

## 下一步
当成功拆解出规律后，进入"隐喻封装"阶段。
