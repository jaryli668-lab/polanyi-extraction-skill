# 波兰尼破壁术 (Polanyi Extraction)

> 把"只可意会"的默会知识提取成"可言说"的内容资产

基于迈克尔·波兰尼的默会知识理论，帮助创作者将那些"烂在肚子里"的经验和直觉，转化为极具价值的内容资产。

## ✨ 核心功能

- **默会雷达捕捉**：拦截"附属意识"，捕捉那些"说不清为什么，但就是知道要这么做"的瞬间
- **降维定格拆解**：像慢动作回放一样，将直觉拆解成可操作的步骤
- **隐喻封装表达**：用人类最古老的协议（故事/类比）打包默会知识
- **质量检验**：确保提取的内容真正有价值，而非 AI 能直接生成的显性知识

## 🎯 适用场景

### 1. 内容创作
- 捕捉创作时的"手感"：为什么这样改标题？为什么这样开篇？
- 拆解成功案例：哪篇内容数据好？当时直觉是什么？

### 2. 工作复盘
- 捕捉关键时刻：危机时刻的直觉判断、搞定难缠客户的下意识反应
- 拗发现决策过程：当时的"第六感"是什么？

### 3. 技能学习
- 捕捉"顿悟时刻"：从不会到会的那一瞬间发生了什么？
- 拆解肌肉记忆：现在你能"自动"做的，但新手不会的动作是什么？

## 🎭 核心理念

**波兰尼悖论**："我们要知道的比我们能能说出的多。"

大部分创作者的误区是：只记录自己"想明白"的**显性知识**（Explicit Knowledge）。而最有价值的知识发生在"没过脑子就做对"的**默会知识**（Tacit Knowledge）瞬间。

本技能通过操控注意力和提取潜意识，进行一场"知识外科手术"。

## 🔍 质量检验标准

完成提取后，通过以下标准检验质量：

1. ✅ **AI 能直接写出来吗？** （如果能，说明全是显性知识，需要重来）
2. ✅ **里面有"伤疤"和"手感"吗？** （痛感、爽感、挫败感...）
3. ✅ **有隐喻做载体吗？** （读者能通过你的隐喻调动自己的身体经验吗？）

## 📚 输出格式

```markdown

## 📍 原始直觉（默会雷达）

## 🔬 降维定格（拆解分析）
### 关键步骤
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

### 深层洞察
[这里写拆解后发现的核心规律]

## 🎯 隐喻封装（可传播表达）

>
> [展开隐喻，让读者能"感受"到你的经验]

## ✅ 质量检验
- [ ] AI 能直接写出来？ → [判断]
- [ ] 有伤疤和手感？ → [判断]
- [ ] 有隐喻载体？ → [判断]

## 💡 可复用的经验
[这里写一句话总结，让别人可以直接使用]
```
